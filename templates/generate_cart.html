Генерация игрового поля будет осуществляться в несколько этапов:
- генерация основного пласта поля;
- изменение части клеток игрового поля под те задачи, которые они должны будут осуществлять.


Для удобства создадим 2 функции: "SpawnPlace" и "BuildOtherPlace"

Игровое поле будет квадратной формы и состоять из клеток (форма которых так же - квадрат) с именами на подобии координат(1;1,1;2...), а значит для его построения нам необходима конструкцию из 2 циклов for.
Заполнять же поле мы будем заранее подготовленными префабами стандартных клеток с травой, а после за счёт функции "BuildOtherPlace" меняем определённый клетки соответствующим образом.


Для клеток дороги это:
- изменение спрайта для всех полей;
- если это первая клетка дороги, добавление объекту скрипта "Spawner";
- если это последняя клетка дороги, добавление объекту тега "end_road";
- если это последняя клетка дороги, добавление объекту скрипта "GetDamageFromEnemy";
- если это последняя клетка дороги, добавление объекту колайдера с включённым свойством "isTrigger".


Для клеток для размещения башен это:
- изменение спрайта клетки;
- установка тега "choice_tower";
- добавление объекту скрипта "CountClicks";
- добавление объекту колайдера, для дальнейшей возможности отследить нажатие игрока на данной поле;
- добавление дочернего объекта(заранее настроенного префаба) с выбором башен и его настройка.


Для правильной работы скрипта 'BuildOtherPlace', что можно увидеть выше необходимо предоставить ему спипок с названиями клеток, подвергающихся изменениям. Список с клетками для размещения башен нам необходимо справится самостоятельно, а вот в случаи с дорогами достаточно будет указать лишь углы необходимой нам дороги.
Но что делать с клетками дороги между углами? Для решения этого вопроса мы напишем ещё одну функцию "FindAllRoad", принемающую на вход список с названиями углов дороги, а на выходе получим список с названием всех клеток, принадлежащих дороге.
